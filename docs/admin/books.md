# 교재 관리법

::: tip
본 메뉴에 접근 가능한 사용자는 `직원`급 이상입니다.
:::

## 교재 파일관리 및 사이드바 관리

현재 교재 파일 관리는 **사이드바 관리 기능과 연결**되어 있습니다.

이 메뉴에서는 크게 두 가지를 할 수 있습니다.

1. **사이드바(왼쪽)에 보일 카테고리와 교재 항목을 관리**
2. **PDF 파일을 업로드하고, 해당 파일을 사이드바 항목과 연결**

---

## 화면 구성 이해하기

![configureSidebar](/images/configureSidebar.png)

### 1. 좌측 미리보기 영역

왼쪽 영역은 실제 사이드바에 가까운 **미리보기**입니다.

- **카테고리(소분류)** 단위로 교재가 묶여 보입니다.
- 각 카테고리를 펼치면 그 안에 연결된 교재 항목들이 표시됩니다.
- 여기서 **수정할 교재 항목을 선택**하면 오른쪽 편집 영역에서 상세 수정이 가능합니다.

---

### 2. 우측 편집 영역

오른쪽 영역은 선택한 카테고리/교재를 수정하는 곳입니다.

---

## 각 항목의 의미

### 대분류 태그

전체 교재를 **태그 기준으로 묶어 접근**하기 위한 정보입니다.

뷰어 페이지에서는 사이드바 목록 위에 태그가 표시되며,  
사용자가 특정 태그를 클릭하면 **해당 태그가 달린 교재만 필터링**해서 볼 수 있습니다.

- 예: `22개정`, `2026` 등
- 현재는 연도별/분류별 교재 기준으로 사전 세팅되어 있습니다.

---

### 카테고리

사이드바에서 교재를 묶는 **분류 이름**입니다.

즉, 왼쪽 사이드바에서 굵게 보이는 묶음 제목이 이 값입니다.

예를 들어 다음과 같은 형태입니다.

- `22개정 세젤쉬`
- `2026 파이널 모의고사 시즌2`

---

### 대분류 순서

카테고리 블록 자체의 순서를 변경하는 기능입니다.

즉, 사이드바에서 카테고리 묶음이 위/아래로 이동합니다.

현재 버튼의 의도는 다음과 같습니다.

- 위로 이동
- 아래로 이동

::: warning
현재 순서 관련 기능은 일부 동작이 직관적이지 않을 수 있어, 추후 수정이 필요합니다.
:::

---

### + 새 소분류

새로운 **카테고리**를 만드는 버튼입니다.

즉, 교재를 담을 새 묶음을 추가할 때 사용합니다.

예:

- `2027 미친개념`
- `2026 파이널 모의고사 시즌3`

---

### 저장 / 변경사항 저장하기

우상단의 **저장** 버튼과 하단의 **변경사항 저장하기** 버튼은 **같은 역할**입니다.

둘 중 어느 버튼을 눌러도 현재 수정한 사이드바 정보를 저장합니다.

---

## 선택한 교재 항목 편집

왼쪽 미리보기에서 교재를 하나 선택하면, 오른쪽에서 아래 항목들을 수정할 수 있습니다.

### 표시 이름

사이드바에서 실제로 보이는 교재 이름입니다.

예:

- `1회`
- `2회`
- `대수`

카테고리 이름만으로도 충분히 설명되는 경우, 표시 이름은 짧게 관리할 수 있습니다.

---

### to

사이드바 항목을 눌렀을 때 연결할 **파일명 기준값**입니다.

이 값은 실제 PDF 파일명과 맞아야 정상적으로 연결됩니다.

즉, 파일 연결에서 가장 우선적으로 참고하는 이름입니다.

---

### id

현재는 `to`가 제대로 동작하지 않을 때를 대비한 **보조 식별값** 성격으로 사용되고 있습니다.

정상적으로 연결이 잘 되는 경우에는 주로 **구분용 값**으로 생각하시면 됩니다.

::: warning
`id`의 정확한 최종 사용 규칙은 추후 추가 확인 후 보완 예정입니다.
:::

---

### 태그(대분류 태그)

상단의 **대분류 태그**와 같은 값입니다.

즉, 현재 선택한 교재가 속한 카테고리의 태그를 수정하는 항목입니다.

---

### 사이드바에서 보이기

오른쪽 끝 스위치로 on/off 할 수 있는 항목입니다.

- **ON**: 사이드바에 표시됨
- **OFF**: 데이터는 유지되지만, 사이드바에서는 보이지 않음

즉, 삭제하지 않고 **숨김 처리**만 하고 싶을 때 사용합니다.

---

### 세부 순서

현재 선택한 교재가 **같은 카테고리 안에서** 몇 번째에 보일지를 조정하는 기능입니다.

버튼의 의도는 다음과 같습니다.

- 맨 앞으로 이동
- 한 칸 위로 이동
- 한 칸 아래로 이동
- 맨 뒤로 이동

::: warning
현재 순서 관련 기능은 일부 동작이 직관적이지 않을 수 있어, 추후 수정이 필요합니다.
:::

---

### 이 항목 삭제하기

이 버튼은 **사이드바에서 해당 교재 항목만 제거**하는 기능입니다.

즉:

- 사이드바 연결은 삭제될 수 있음
- 실제 PDF 파일 자체가 함께 삭제되는 기능은 아닌 것으로 보임

## PDF 업로드

### PDF 업로드 버튼의 역할

`PDF 업로드` 버튼은 실제 PDF 파일을 업로드하는 기능입니다.

업로드가 끝나면 저장도 바로 반영됩니다.

다만, **PDF를 업로드했다고 해서 자동으로 사이드바에 바로 노출되지는 않습니다.**

업로드 후에는 별도로 **새 교재 항목과 연결**해야 실제 사이드바에 나타납니다.

즉, 흐름은 다음과 같습니다.

1. PDF 업로드
2. 업로드 완료
3. 새 교재 항목과 연결
4. 사이드바에서 노출 시작

::: tip
정리하면, PDF 업로드는 “파일 등록”이고,  
사이드바 교재 항목 추가는 “화면에 보이게 연결”하는 작업입니다.
:::

---

## 사이드바에 교재 항목 추가

### + 사이드바에 교재 항목 추가

이 버튼은 **데이터베이스에 있는 파일을 사이드바 항목으로 연결하여 목록에 추가**하는 기능입니다.

즉, 이미 존재하는 교재 파일을 사이드바에 노출시키기 위한 기능입니다.

입력하게 되는 정보는 다음과 같습니다.

- **표시 이름**: 사이드바에서 보일 이름
- **to**: 연결할 파일명 기준값
- **id**: 보조 식별값

::: warning
구체적인 “교재 추가 절차”는 기능 수정 후 별도 안내 예정입니다.
:::

---

## 일반적인 작업 흐름

### 새 카테고리를 만들고 교재를 관리할 때

1. `+ 새 소분류`로 카테고리를 생성합니다.
2. 필요하면 카테고리의 **대분류 태그**, **카테고리명**, **표시 여부**를 조정합니다.
3. PDF 파일을 업로드합니다.
4. 업로드된 파일을 사이드바 교재 항목과 연결합니다.
5. 필요한 경우 교재의 표시 이름, 순서 등을 수정합니다.
6. `저장` 또는 `변경사항 저장하기`를 눌러 반영합니다.

---

## 자주 헷갈리는 점

### Q. PDF만 업로드하면 사용자에게 바로 보이나요?

아니요.

PDF 업로드는 파일 등록일 뿐이며,  
**사이드바 교재 항목으로 연결해야** 실제 화면에 보이기 시작합니다.

---

### Q. 사이드바에서 보이기 OFF는 삭제인가요?

아니요.

데이터는 남아 있고, 사이드바에서만 숨겨집니다.

---

### Q. 우상단 저장과 하단 저장은 다른 기능인가요?

아니요. 같은 저장 기능입니다.

---

### Q. 이 항목 삭제하기를 누르면 PDF도 같이 삭제되나요?

현재 기준으로는 **사이드바에서만 삭제**되는 것으로 보입니다.
